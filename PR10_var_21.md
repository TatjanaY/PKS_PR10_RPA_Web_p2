## 📖 Ситуация

После запуска базового портала регистрации бронирований компания **LibraryBook Pro** столкнулась с необходимостью управления жизненным циклом выдачи книг:

* необходимо выдавать книги после подтверждения бронирования;
* отмечать возврат книг читателями;
* удалять неактуальные или ошибочные записи.

В связи с этим было принято решение:

* расширить функциональность веб-приложения;
* реализовать более сложный сценарий автоматизации с использованием **Puzzle RPA**.

## 🔘 Новые кнопки управления

---

### `Issue Book`

**Назначение:**
Выдача книги читателю

**Логика:**

```json
{ "status": "Issued" }
```

**Требования:**

* доступна только если `Status ≠ "Issued"`
* при нажатии изменяет статус выбранной записи на `"Issued"`

---

### `Return Book`

**Назначение:**
Отметка возврата книги в библиотеку

**Логика:**

```json
{ "status": "Returned" }
```

**Требования:**

* доступна только если `Status = "Issued"`
* при нажатии изменяет статус выбранной записи на `"Returned"`

---

### `Delete Reservation`

**Назначение:**
Удаление бронирования из системы

**Логика:**

* удаляет выбранную запись из таблицы

**Требования:**

* доступна только если выбрана строка таблицы
* после удаления таблица обновляется

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

### Кнопки `Issue Book`, `Return Book`, `Delete Reservation`

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

* `Issue Book` должна быть неактивна, если:

```json
{ "status": "Issued" }
```

* `Return Book` должна быть неактивна, если:

```json
{ "status": "Returned" }
```

## 🔘 Детализация кнопок управления

---

### `Issue Book`

**Назначение:**
Выдача книги читателю после подтверждения бронирования.

**Бизнес-смысл:**
Книга зарезервирована и выдана читателю.

---

**Когда используется:**

* бронирование подтверждено;
* книга доступна в фонде.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус бронирования:

   * если `Status = Issued` → операция запрещена;
   * если `Status ≠ Issued` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "Issued" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно выдать уже обработанное бронирование;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `Issued`.

---

### `Return Book`

**Назначение:**
Отметка возврата книги в библиотеку.

**Бизнес-смысл:**
Книга возвращена читателем и доступна для следующей выдачи.

---

**Когда используется:**

* читатель вернул книгу в срок;
* книга проверена на целостность.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается только `Issued`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Returned" }
```

4. Таблица обновляется.

---

**Ограничения:**

* нельзя вернуть книгу, которая не была выдана;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже `Returned`.

---

### `Delete Reservation`

**Назначение:**
Удаление бронирования из системы.

**Бизнес-смысл:**
Бронирование более не актуально, запись удаляется.

---

**Когда используется:**

* читатель отменил бронирование;
* запись была создана ошибочно.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. При нажатии:

   * запись удаляется из таблицы;
3. Таблица обновляется.

---

**Ограничения:**

* кнопка `disabled`, если:

  * нет выбранной строки;
* после удаления интерфейс корректно обновляется.

## 🔍 Детализация фильтрации `Filter by Category`

---

### 📌 Назначение

Фильтр позволяет отображать бронирования в зависимости от категории книги.

---

## 📊 Значения категорий и их смысл

| Категория | Значение | Когда используется |
| --------- | -------- | ------------------ |
| `Programming` | Книги по программированию | заявки на изучение языков |
| `Databases` | Книги по базам данных | заявки на проектирование БД |
| `Networking` | Книги по сетевым технологиям | заявки на изучение сетей |

---

## ⚙️ Алгоритм работы фильтра

---

### 1. Выбор значения

Пользователь выбирает значение в выпадающем списке:

```
Filter by Category
```

---

### 2. Обработка выбора

#### Если выбрано:

### `All`

* отображаются все записи без фильтрации

---

### `Programming`

* отображаются только записи:

```json
{ "category": "Programming" }
```

---

### `Databases`

* отображаются только:

```json
{ "category": "Databases" }
```

---

### `Networking`

* отображаются только:

```json
{ "category": "Networking" }
```

---

### 3. Применение фильтра

* фильтрация выполняется:

  * без перезагрузки страницы;
  * на стороне клиента (или сервера — опционально);
* обновление таблицы происходит с задержкой (имитация реального API).

---

### 4. Поведение при отсутствии данных

Если записи отсутствуют:

* отображается сообщение в текстовой метке:

```
No reservations found
```

---

### 5. Взаимодействие с другими элементами

---

#### ✔ После изменения статуса

* запись может "исчезнуть" из текущего фильтра
  (например, была `Reserved`, стала `Issued`)

---

#### ✔ После возврата

* таблица автоматически обновляется с учётом текущего фильтра

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий бронирования в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление бронирований

Для каждой записи:

* заполнить форму
* проверить `Add Reservation.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Выдача книг категории Programming

* найти бронирования с `Category = Programming`

* для каждой найденной записи:

  * выбрать строку
  * проверить доступность `Issue Book`
  * изменить статус на:

```json
{ "status": "Issued" }
```

---

### 4. Возврат одной книги

* выбрать одну запись со статусом `Issued`
* нажать `Return Book`
* убедиться, что статус изменился на `"Returned"`

---

### 5. Удаление записи

* выбрать любую запись
* нажать `Delete Reservation`
* убедиться, что запись исчезла

---

### 6. Использование фильтра

* выбрать `Filter by Category = Programming`
* убедиться, что отображаются только нужные записи

---

### 7. Поиск

* выполнить поиск по имени читателя (например: `"Ivan"`)
* проверить корректность результата

---

### 8. Проверка итогового состояния

* проверить:

  * все бронирования добавлены (10 записей)
  * книги категории `Programming` имеют статус `"Issued"`
  * одна запись возвращена
  * одна запись удалена
