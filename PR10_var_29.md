---

## 📖 Ситуация

После запуска базового портала регистрации компания **InternalTraining Pro** столкнулась с необходимостью управления жизненным циклом заявок:

* необходимо подтверждать участие после проверки доступности мест;
* отменять неактуальные или ошибочные регистрации;
* удалять неактуальные или ошибочные записи.

В связи с этим было принято решение:

* расширить функциональность веб-приложения;
* реализовать более сложный сценарий автоматизации с использованием **Puzzle RPA**.

## 🔘 Новые кнопки управления

---

### `Confirm Participation`

**Назначение:**
Подтверждение участия в тренинге

**Логика:**

```json
{ "status": "Confirmed" }
```

**Требования:**

* доступна только если `Status = "Registered"`
* при нажатии изменяет статус выбранной записи на `"Confirmed"`

---

### `Cancel Participation`

**Назначение:**
Отмена участия в тренинге

**Логика:**

```json
{ "status": "Cancelled" }
```

**Требования:**

* доступна только если `Status = "Registered"`
* при нажатии изменяет статус выбранной записи на `"Cancelled"`

---

### `Delete Registration`

**Назначение:**
Удаление регистрации из системы

**Логика:**

* удаляет выбранную запись из таблицы

**Требования:**

* доступна только если выбрана строка таблицы
* после удаления таблица обновляется

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

### Кнопки `Confirm Participation`, `Cancel Participation`, `Delete Registration`

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

* `Confirm Participation` должна быть неактивна, если:

```json
{ "status": "Confirmed" }
или
{ "status": "Cancelled" }
```

* `Cancel Participation` должна быть неактивна, если:

```json
{ "status": "Confirmed" }
или
{ "status": "Cancelled" }
```

## 🔘 Детализация кнопок управления

---

### `Confirm Participation`

**Назначение:**
Подтверждение участия после проверки доступности мест.

**Бизнес-смысл:**
Заявка проверена, сотрудник допущен к тренингу.

---

**Когда используется:**

* администратор проверил корректность данных;
* есть свободные места на тренинге.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус регистрации:

   * если `Status ≠ Registered` → операция запрещена;
   * если `Status = Registered` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "Confirmed" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно подтвердить уже обработанную заявку;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `Confirmed` или `Cancelled`.

---

### `Cancel Participation`

**Назначение:**
Отмена участия в тренинге.

**Бизнес-смысл:**
Заявка более не актуальна, место освобождается.

---

**Когда используется:**

* сотрудник отменил участие;
* регистрация была создана ошибочно.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается только `Registered`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Cancelled" }
```

4. Таблица обновляется.

---

**Ограничения:**

* повторная отмена невозможна;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже обработан.

---

### `Delete Registration`

**Назначение:**
Удаление регистрации из системы.

**Бизнес-смысл:**
Регистрация более не актуальна, запись удаляется.

---

**Когда используется:**

* сотрудник отменил заявку;
* запись была создана ошибочно.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. При нажатии:

   * запись удаляется из таблицы;
3. Таблица обновляется.

---

**Ограничения:**

* кнопка `disabled`, если:

  * нет выбранной строки;
* после удаления интерфейс корректно обновляется.

## 🔍 Детализация фильтрации `Filter by Training`

---

### 📌 Назначение

Фильтр позволяет отображать регистрации в зависимости от темы тренинга.

---

## 📊 Значения тренингов и их смысл

| Тренинг | Значение | Когда используется |
| ------- | -------- | ------------------ |
| `Project Management` | Управление проектами | заявки на изучение методологий |
| `DevOps` | DevOps-практики | заявки на автоматизацию |
| `Data Analysis` | Анализ данных | заявки на аналитику |

---

## ⚙️ Алгоритм работы фильтра

---

### 1. Выбор значения

Пользователь выбирает значение в выпадающем списке:

```
Filter by Training
```

---

### 2. Обработка выбора

#### Если выбрано:

### `All`

* отображаются все записи без фильтрации

---

### `Project Management`

* отображаются только записи:

```json
{ "training": "Project Management" }
```

---

### `DevOps`

* отображаются только:

```json
{ "training": "DevOps" }
```

---

### `Data Analysis`

* отображаются только:

```json
{ "training": "Data Analysis" }
```

---

### 3. Применение фильтра

* фильтрация выполняется:

  * без перезагрузки страницы;
  * на стороне клиента (или сервера — опционально);
* обновление таблицы происходит с задержкой (имитация реального API).

---

### 4. Поведение при отсутствии данных

Если записи отсутствуют:

* отображается сообщение в текстовой метке:

```
No registrations found
```

---

### 5. Взаимодействие с другими элементами

---

#### ✔ После изменения статуса

* запись может "исчезнуть" из текущего фильтра
  (например, была `Registered`, стала `Confirmed`)

---

#### ✔ После удаления

* таблица автоматически обновляется с учётом текущего фильтра

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий регистрации в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление регистраций

Для каждой записи:

* заполнить форму
* проверить `Register.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Подтверждение участия в тренинге DevOps

* найти регистрации с `Training Topic = DevOps`

* для каждой найденной записи:

  * выбрать строку
  * проверить доступность `Confirm Participation`
  * изменить статус на:

```json
{ "status": "Confirmed" }
```

---

### 4. Использование фильтра

* выбрать `Filter by Training = DevOps`
* убедиться, что отображаются только нужные записи

---

### 5. Поиск

* выполнить поиск по имени сотрудника (например: `"Ivan"`)
* проверить корректность результата

---

### 6. Отмена одной регистрации

* выбрать любую запись со статусом `Registered`
* нажать `Cancel Participation`
* убедиться, что статус изменился на `"Cancelled"`

---

### 7. Проверка итогового состояния

* проверить:

  * все регистрации добавлены (10 записей)
  * регистрации тренинга `DevOps` имеют статус `"Confirmed"`
  * одна регистрация отменена
