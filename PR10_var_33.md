## 📖 Ситуация

После запуска базового портала регистрации бронирований компания **EduSpace Pro** столкнулась с необходимостью управления жизненным циклом заявок:

* необходимо подтверждать бронирования после проверки расписания;
* отменять неактуальные или ошибочные заявки;
* удалять неактуальные или ошибочные записи.

В связи с этим было принято решение:

* расширить функциональность веб-приложения;
* реализовать более сложный сценарий автоматизации с использованием **Puzzle RPA**.

## 🔘 Новые кнопки управления

---

### `Confirm Booking`

**Назначение:**
Подтверждение бронирования аудитории

**Логика:**

```json
{ "status": "Confirmed" }
```

**Требования:**

* доступна только если `Status = "Booked"`
* при нажатии изменяет статус выбранной записи на `"Confirmed"`

---

### `Cancel Booking`

**Назначение:**
Отмена бронирования аудитории

**Логика:**

```json
{ "status": "Cancelled" }
```

**Требования:**

* доступна только если `Status = "Booked"`
* при нажатии изменяет статус выбранной записи на `"Cancelled"`

---

### `Delete Record`

**Назначение:**
Удаление записи из системы

**Логика:**

* удаляет выбранную запись из таблицы

**Требования:**

* доступна только если выбрана строка таблицы
* после удаления таблица обновляется

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

```json
{ "status": "Confirmed" }

```

* `Cancel Booking` должна быть неактивна, если:

```json
{ "status": "Confirmed" }
или
{ "status": "Cancelled" }
```

## 🔘 Детализация кнопок управления

---

### `Confirm Booking`

**Назначение:**
Подтверждение бронирования после проверки расписания.

**Бизнес-смысл:**
Заявка проверена, аудитория зарезервирована за преподавателем.

---

**Когда используется:**

* администратор проверил отсутствие конфликтов;
* бронирование соответствует расписанию.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус бронирования:

   * если `Status ≠ Booked` → операция запрещена;
   * если `Status = Booked` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "Confirmed" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно подтвердить уже обработанную заявку;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `Confirmed` или `Cancelled`.

---

### `Cancel Booking`

**Назначение:**
Отмена бронирования аудитории.

**Бизнес-смысл:**
Заявка более не актуальна, аудитория освобождается.

---

**Когда используется:**

* преподаватель отменил занятие;
* бронирование было создано ошибочно.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается только `Booked`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Cancelled" }
```

4. Таблица обновляется.

---

**Ограничения:**

* повторная отмена невозможна;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже обработан.

---

### `Delete Record`

**Назначение:**
Удаление записи из системы.

**Бизнес-смысл:**
Запись более не актуальна, удаляется из базы.

---

**Когда используется:**

* запись была создана ошибочно;
* требуется очистка тестовых данных.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. При нажатии:

   * запись удаляется из таблицы;
3. Таблица обновляется.

---

**Ограничения:**

* кнопка `disabled`, если:

  * нет выбранной строки;
* после удаления интерфейс корректно обновляется.

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий бронирования в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление бронирований

Для каждой записи:

* заполнить форму
* проверить `Book Room.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Подтверждение бронирований утреннего слота

* найти бронирования с `Time Slot = Morning`

* для каждой найденной записи:

  * выбрать строку
  * проверить доступность `Confirm Booking`
  * изменить статус на:

```json
{ "status": "Confirmed" }
```

---

### 4. Отмена двух бронирований

* выбрать две записи со статусом `Booked`
* нажать `Cancel Booking` для каждой
* убедиться, что статус изменился на `"Cancelled"`

---

### 5. Удаление записи

* выбрать любую запись
* нажать `Delete Record`
* убедиться, что запись исчезла

---

### 6. Использование фильтра

* выбрать `Filter by Day = Monday`
* убедиться, что отображаются только нужные записи

---

### 7. Проверка итогового состояния

* проверить:

  * все бронирования добавлены (10 записей)
  * бронирования слота `Morning` имеют статус `"Confirmed"`
  * две записи отменены
  * одна запись удалена
