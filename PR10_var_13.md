## 📖 Ситуация

После запуска базового портала регистрации доставок компания **InternalLogistics Pro** столкнулась с необходимостью управления жизненным циклом отправок:

* необходимо начинать доставку после регистрации заявки;
* отмечать завершение доставки после получения посылки;
* отслеживать статус выполнения отправок.

В связи с этим было принято решение:

* расширить функциональность веб-приложения;
* реализовать более сложный сценарий автоматизации с использованием **Puzzle RPA**.

# **Задание 1. Расширение функциональности веб-приложения**

Необходимо доработать существующее веб-приложение, добавив новые элементы управления и логику обработки доставок.

**Каждый элемент интерфейса должен иметь уникальный `id`**

## 🔘 Новые кнопки управления

---

### `Dispatch Package`

**Назначение:**
Начало доставки посылки

**Логика:**

```json
{ "status": "In Delivery" }
```

**Требования:**

* доступна только если `Status = "Created"`
* при нажатии изменяет статус выбранной записи на `"In Delivery"`

---

### `Mark Delivered`

**Назначение:**
Отметка о получении посылки

**Логика:**

```json
{ "status": "Delivered" }
```

**Требования:**

* доступна только если `Status = "In Delivery"`
* при нажатии изменяет статус выбранной записи на `"Delivered"`

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

### Кнопки `Dispatch Package`, `Mark Delivered`

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

* `Dispatch Package` должна быть неактивна, если:

```json
{ "status": "In Delivery" }
или
{ "status": "Delivered" }
```

* `Mark Delivered` должна быть неактивна, если:

```json
{ "status": "Created" }
или
{ "status": "Delivered" }
```

## 🔘 Детализация кнопок управления

---

### `Dispatch Package`

**Назначение:**
Начало доставки после регистрации заявки.

**Бизнес-смысл:**
Курьер приступил к транспортировке посылки.

---

**Когда используется:**

* заявка зарегистрирована и проверена;
* курьер получил задание на доставку.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус доставки:

   * если `Status ≠ Created` → операция запрещена;
   * если `Status = Created` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "In Delivery" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно отправить уже доставляемую посылку;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `In Delivery` или `Delivered`.

---

### `Mark Delivered`

**Назначение:**
Отметка о получении посылки получателем.

**Бизнес-смысл:**
Посылка доставлена, заявка закрыта.

---

**Когда используется:**

* курьер передал посылку получателю;
* получатель подтвердил получение.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается только `In Delivery`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Delivered" }
```

4. Таблица обновляется.

---

**Ограничения:**

* нельзя отметить доставку, которая не была начата;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже обработан.

## 🔍 Фильтрация доставок

Необходимо добавить блок фильтрации:

### `Filter by Priority`

**Значения:**

* `All`
* `Low`
* `Medium`
* `High`

---

### Логика

* при выборе значения отображаются только соответствующие записи
* фильтрация выполняется без перезагрузки страницы

---

## 🔍 Детализация фильтрации `Filter by Priority`

---

### 📌 Назначение

Фильтр позволяет отображать доставки в зависимости от приоритета.

---

## 📊 Значения приоритетов и их смысл

| Приоритет | Значение | Когда назначается |
| --------- | -------- | ----------------- |
| `Low` | Низкий приоритет | при стандартных отправлениях |
| `Medium` | Средний приоритет | при срочных документах |
| `High` | Высокий приоритет | при критических грузах |

---

## ⚙️ Алгоритм работы фильтра

---

### 1. Выбор значения

Пользователь выбирает значение в выпадающем списке:

```
Filter by Priority
```

---

### 2. Обработка выбора

#### Если выбрано:

### `All`

* отображаются все записи без фильтрации

---

### `Low`

* отображаются только записи:

```json
{ "priority": "Low" }
```

---

### `Medium`

* отображаются только:

```json
{ "priority": "Medium" }
```

---

### `High`

* отображаются только:

```json
{ "priority": "High" }
```

---

### 3. Применение фильтра

* фильтрация выполняется:

  * без перезагрузки страницы;
  * на стороне клиента (или сервера — опционально);
* обновление таблицы происходит с задержкой (имитация реального API).

---

### 4. Поведение при отсутствии данных

Если записи отсутствуют:

* отображается сообщение в текстовой метке:

```
No deliveries found
```

---

### 5. Взаимодействие с другими элементами

---

#### ✔ После изменения статуса

* запись может "исчезнуть" из текущего фильтра
  (например, была `Created`, стала `Delivered`)

---

#### ✔ После удаления

* таблица автоматически обновляется с учётом текущего фильтра

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий доставки в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление доставок

Для каждой записи:

* заполнить форму
* проверить `Create Delivery.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Начало доставки всех заявок

После добавления:

* для каждой записи в таблице:

  * выбрать строку
  * проверить доступность `Dispatch Package`
  * изменить статус на:

```json
{ "status": "In Delivery" }
```

---

### 4. Отметка доставленных посылок с высоким приоритетом

* найти доставки с `Priority = High`

* для каждой найденной записи:

  * выбрать строку
  * проверить доступность `Mark Delivered`
  * изменить статус на:

```json
{ "status": "Delivered" }
```

---

### 5. Использование фильтра

* выбрать `Filter by Priority = High`
* убедиться, что отображаются только нужные записи

---

### 6. Поиск

* выполнить поиск по описанию (например: `"Contract"`)
* проверить корректность результата

---

### 7. Проверка итогового состояния

* проверить:

  * все доставки добавлены (10 записей)
  * все записи имеют статус `"In Delivery"` или `"Delivered"`
  * доставки с `Priority = High` имеют статус `"Delivered"`
