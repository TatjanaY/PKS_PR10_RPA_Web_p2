## 📖 Ситуация

После запуска базового портала регистрации заявок компания **LicenseManager Pro** столкнулась с необходимостью управления жизненным циклом лицензий:

* необходимо активировать лицензии после проверки заявок;
* отзывать неактуальные или просроченные лицензии;
* отслеживать статус выполнения заявок.

В связи с этим было принято решение:

* расширить функциональность веб-приложения;
* реализовать более сложный сценарий автоматизации с использованием **Puzzle RPA**.

# **Задание 1. Расширение функциональности веб-приложения**

Необходимо доработать существующее веб-приложение, добавив новые элементы управления и логику обработки заявок.

**Каждый элемент интерфейса должен иметь уникальный `id`**

---

## 🔘 Новые кнопки управления

---

### `Activate License`

**Назначение:**
Активация лицензии для сотрудника

**Логика:**

```json
{ "status": "Active" }
```

**Требования:**

* доступна только если `Status = "Requested"`
* при нажатии изменяет статус выбранной записи на `"Active"`

---

### `Revoke License`

**Назначение:**
Отзыв лицензии

**Логика:**

```json
{ "status": "Revoked" }
```

**Требования:**

* доступна только если `Status ≠ "Revoked"`
* при нажатии изменяет статус выбранной записи на `"Revoked"`

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

### Кнопки `Activate License`, `Revoke License`

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

* `Activate License` должна быть неактивна, если:

```json
{ "status": "Active" }
или
{ "status": "Revoked" }
```

* `Revoke License` должна быть неактивна, если:

```json
{ "status": "Revoked" }
```

## 🔘 Детализация кнопок управления

---

### `Activate License`

**Назначение:**
Активация лицензии после проверки заявки.

**Бизнес-смысл:**
Лицензия проверена и активирована для сотрудника.

---

**Когда используется:**

* администратор проверил обоснование запроса;
* лицензия доступна в пуле.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус заявки:

   * если `Status ≠ Requested` → операция запрещена;
   * если `Status = Requested` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "Active" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно активировать уже обработанную заявку;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `Active` или `Revoked`.

---

### `Revoke License`

**Назначение:**
Отзыв лицензии у сотрудника.

**Бизнес-смысл:**
Лицензия более не требуется или срок действия истёк.

---

**Когда используется:**

* сотрудник завершил проект;
* лицензия просрочена.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается любой статус кроме `Revoked`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Revoked" }
```

4. Таблица обновляется.

---

**Ограничения:**

* нельзя отозвать уже отозванную лицензию;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже `Revoked`.

## 🔍 Фильтрация заявок

Необходимо добавить блок фильтрации:

### `Filter by License Type`

**Значения:**

* `All`
* `Trial`
* `Standard`
* `Enterprise`

---

### Логика

* при выборе значения отображаются только соответствующие записи
* фильтрация выполняется без перезагрузки страницы

---

## 🔍 Детализация фильтрации `Filter by License Type`

---

### 📌 Назначение

Фильтр позволяет отображать заявки в зависимости от типа лицензии.

---

## 📊 Значения типов лицензий и их смысл

| Тип лицензии | Значение | Когда используется |
| ------------ | -------- | ------------------ |
| `Trial` | Пробная лицензия | для тестирования ПО |
| `Standard` | Стандартная лицензия | для рядовых сотрудников |
| `Enterprise` | Корпоративная лицензия | для ключевых специалистов |

---

## ⚙️ Алгоритм работы фильтра

---

### 1. Выбор значения

Пользователь выбирает значение в выпадающем списке:

```
Filter by License Type
```

---

### 2. Обработка выбора

#### Если выбрано:

### `All`

* отображаются все записи без фильтрации

---

### `Trial`

* отображаются только записи:

```json
{ "license_type": "Trial" }
```

---

### `Standard`

* отображаются только:

```json
{ "license_type": "Standard" }
```

---

### `Enterprise`

* отображаются только:

```json
{ "license_type": "Enterprise" }
```

---

### 3. Применение фильтра

* фильтрация выполняется:

  * без перезагрузки страницы;
  * на стороне клиента (или сервера — опционально);
* обновление таблицы происходит с задержкой (имитация реального API).

---

### 4. Поведение при отсутствии данных

Если записи отсутствуют:

* отображается сообщение в текстовой метке:

```
No license requests found
```

---

### 5. Взаимодействие с другими элементами

---

#### ✔ После изменения статуса

* запись может "исчезнуть" из текущего фильтра
  (например, была `Requested`, стала `Active`)

---

#### ✔ После отзыва

* таблица автоматически обновляется с учётом текущего фильтра

---

## 🎯 Итоговая логика состояний

Подача заявки → Requested
           ↓
          Active
           ↓
         Revoked

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий заявки в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление заявок

Для каждой записи:

* заполнить форму
* проверить `Request License.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Активация всех лицензий

После добавления:

* для каждой записи в таблице:

  * выбрать строку
  * проверить доступность `Activate License`
  * изменить статус на:

```json
{ "status": "Active" }
```

---

### 4. Отзыв одной лицензии

* выбрать любую запись
* нажать `Revoke License`
* убедиться, что статус изменился на `"Revoked"`

---

### 5. Использование фильтра

* выбрать `Filter by License Type = Enterprise`
* убедиться, что отображаются только нужные записи

---

### 6. Поиск

* выполнить поиск по названию ПО (например: `"IntelliJ"`)
* проверить корректность результата

---

### 7. Проверка итогового состояния

* проверить:

  * все заявки добавлены (10 записей)
  * все записи имеют статус `"Active"` или `"Revoked"`
  * одна запись отозвана
