# **Задание 1. Расширение функциональности веб-приложения**

**Каждый элемент интерфейса должен иметь уникальный `id`**

---

## 🔘 Новые кнопки управления

---

### `Start Task`

**Назначение:**
Начало выполнения задачи

**Логика:**

```json
{ "status": "In Progress" }
```

**Требования:**

* доступна только если `Status = "New"`
* при нажатии изменяет статус выбранной записи на `"In Progress"`

---

### `Complete Task`

**Назначение:**
Завершение выполнения задачи

**Логика:**

```json
{ "status": "Done" }
```

**Требования:**

* доступна только если `Status = "In Progress"`
* при нажатии изменяет статус выбранной записи на `"Done"`

---

## ⚙️ Управление состоянием элементов (`disabled`)

---

### Кнопки `Start Task`, `Complete Task`

Должны быть:

* `disabled = true`, если:

  * ни одна строка не выбрана

* `disabled = false`, если:

  * выбрана строка таблицы

---

### Дополнительные условия

* `Start Task` должна быть неактивна, если:

```json
{ "status": "In Progress" }
или
{ "status": "Done" }
```

* `Complete Task` должна быть неактивна, если:

```json
{ "status": "New" }
или
{ "status": "Done" }
```

## 🔘 Детализация кнопок управления

---

### `Start Task`

**Назначение:**
Начало выполнения задачи после распределения.

**Бизнес-смысл:**
Разработчик приступил к реализации задачи.

---

**Когда используется:**

* задача распределена исполнителю;
* определены требования и сроки.

---

**Алгоритм работы:**

1. Пользователь выбирает строку в таблице.
2. Проверяется текущий статус задачи:

   * если `Status ≠ New` → операция запрещена;
   * если `Status = New` → операция разрешена.
3. При нажатии кнопки:

   * статус изменяется на:

```json
{ "status": "In Progress" }
```

4. Таблица обновляется (с задержкой, если реализована асинхронность).

---

**Ограничения:**

* нельзя повторно начать уже выполняемую задачу;
* кнопка должна быть `disabled`, если:

  * не выбрана строка;
  * статус уже `In Progress` или `Done`.

---

### `Complete Task`

**Назначение:**
Завершение выполнения задачи.

**Бизнес-смысл:**
Функционал реализован, задача готова к тестированию.

---

**Когда используется:**

* разработчик завершил реализацию;
* проведено самопроверка кода.

---

**Алгоритм работы:**

1. Выбирается строка таблицы.
2. Проверяется статус:

   * допускается только `In Progress`.
3. При нажатии:

   * статус изменяется на:

```json
{ "status": "Done" }
```

4. Таблица обновляется.

---

**Ограничения:**

* нельзя завершить задачу, которая не была начата;
* кнопка `disabled`, если:

  * нет выбранной строки;
  * статус уже обработан.

## 🔍 Фильтрация задач

Необходимо добавить блок фильтрации:

### `Filter by Project`

**Значения:**

* `All`
* `Web Portal`
* `Mobile App`
* `Internal System`

---

### Логика

* при выборе значения отображаются только соответствующие записи
* фильтрация выполняется без перезагрузки страницы

---

## 🔍 Детализация фильтрации `Filter by Project`

---

### 📌 Назначение

Фильтр позволяет отображать задачи в зависимости от проекта.

---

## 📊 Значения проектов и их смысл

| Проект | Значение | Когда используется |
| ------ | -------- | ------------------ |
| `Web Portal` | Веб-портал компании | задачи по фронтенду и бэкенду |
| `Mobile App` | Мобильное приложение | задачи по iOS/Android |
| `Internal System` | Внутренняя система | задачи по инфраструктуре |

---

## ⚙️ Алгоритм работы фильтра

---

### 1. Выбор значения

Пользователь выбирает значение в выпадающем списке:

```
Filter by Project
```

---

### 2. Обработка выбора

#### Если выбрано:

### `All`

* отображаются все записи без фильтрации

---

### `Web Portal`

* отображаются только записи:

```json
{ "project": "Web Portal" }
```

---

### `Mobile App`

* отображаются только:

```json
{ "project": "Mobile App" }
```

---

### `Internal System`

* отображаются только:

```json
{ "project": "Internal System" }
```

---

### 3. Применение фильтра

* фильтрация выполняется:

  * без перезагрузки страницы;
  * на стороне клиента (или сервера — опционально);
* обновление таблицы происходит с задержкой (имитация реального API).

---

### 4. Поведение при отсутствии данных

Если записи отсутствуют:

* отображается сообщение в текстовой метке:

```
No tasks found
```

---

### 5. Взаимодействие с другими элементами

---

#### ✔ После изменения статуса

* запись может "исчезнуть" из текущего фильтра
  (например, была `New`, стала `Done`)

---

#### ✔ После удаления

* таблица автоматически обновляется с учётом текущего фильтра

# **Задание 2. Разработка продвинутого RPA-сценария**

Необходимо реализовать процесс автоматизации, обрабатывающий задачи в веб-приложении.

---

## 🤖 Алгоритм работы робота

### 1. Подготовка

* открыть браузер
* перейти на сайт
* дождаться загрузки страницы

---

### 2. Добавление задач

Для каждой записи:

* заполнить форму
* проверить `Add Task.disabled`
* при необходимости исправить данные
* отправить форму
* дождаться сообщения об успехе в текстовой метке

---

### 3. Начало выполнения всех задач

После добавления:

* для каждой записи в таблице:

  * выбрать строку
  * проверить доступность `Start Task`
  * изменить статус на:

```json
{ "status": "In Progress" }
```

---

### 4. Завершение задач с высоким приоритетом

* найти задачи с `Priority = High`

* для каждой найденной задачи:

  * выбрать строку
  * проверить доступность `Complete Task`
  * изменить статус на:

```json
{ "status": "Done" }
```

---

### 5. Использование фильтра

* выбрать `Filter by Project = Web Portal`
* убедиться, что отображаются только нужные записи

---

### 6. Поиск

* выполнить поиск по названию задачи (например: `"login"`)
* проверить корректность результата

---

### 7. Проверка итогового состояния

* проверить:

  * все задачи добавлены (10 записей)
  * все записи имеют статус `"In Progress"` или `"Done"`
  * задачи с `Priority = High` имеют статус `"Done"`
